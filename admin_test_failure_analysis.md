# Admin 模块测试用例失败分析报告

## 总体情况
- **总计**: 43 个测试用例
- **通过**: 29 个 ✅
- **失败**: 14 个 ❌
- **通过率**: 67.4%

## 失败用例分类分析

### 一、用户状态相关（2 个）

#### 1. TC-ADMIN-003_筛选已审核
**错误**: `AssertionError: assert False is True`
**原因**: 测试预期返回的用户都是 `is_verified=True`，但实际返回了未认证用户
**分析**: 
- 后端 `/api/users/admin/users/?type=verified` 接口可能没有正确过滤
- 或者测试数据中有用户状态不一致

#### 2. TC-ADMIN-004_非管理员访问
**错误**: `AssertionError: 状态码不匹配：期望 403, 实际 200`
**原因**: 使用 student 账号访问 admin 接口，应该返回 403，但返回了 200
**分析**:
- 后端权限验证可能有问题
- 或者 student 账号意外获得了 admin 权限

### 二、用户审核操作（4 个）

#### 3. TC-ADMIN-005_通过审核

**原因**: 后端接口返回 404，说明接口路径或参数有问题

#### 4. TC-ADMIN-006_拒绝审核  
**错误**: `AssertionError: 状态码不匹配：期望 200, 实际 404`
**原因**: 同上，后端接口返回 404

#### 5. TC-ADMIN-007_用户不存在
**错误**: `AssertionError: message 字段不匹配：期望包含'用户不存在', 实际为''`
**原因**: 后端返回的错误消息为空
**分析**: 后端验证逻辑可能不完善

#### 6. TC-ADMIN-008_无效操作
**错误**: `AssertionError: message 字段不匹配：期望包含'无效的操作', 实际为''`
**原因**: 后端返回的错误消息为空

#### 10. TC-WISH-004_非管理员访问心愿列表

### 五、心愿审核操作（3 个）

#### 11. TC-WISH-MANAGE-003_wish_id 为空
**错误**: `AssertionError: 状态码不匹配：期望 400, 实际 404`
**原因**: 后端返回 404 而不是 400
**分析**: 可能是接口路径构建错误

#### 12. TC-WISH-MANAGE-004_action 为空
**错误**: `AssertionError: 状态码不匹配：期望 400, 实际 500`

**分析**: 后端代码可能有 bug，没有正确处理空 action 参数

**错误**: `AssertionError: 状态码不匹配：期望 400, 实际 200`
