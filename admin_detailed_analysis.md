## 失败用例详细分析

### 一、用户管理模块 (2 个失败)

#### 1. TC-ADMIN-003_筛选已审核
**错误类型**: `AssertionError: assert False is True`  
**失败原因**: 
```python
assert user.get('is_verified') is True
```
测试预期返回的用户都是 `is_verified=True`，但实际返回了未认证用户。

**问题分析**:
- 后端 `/api/users/admin/users/?type=verified` 接口没有正确过滤
- 数据库中可能没有已认证的普通用户数据
- 或者查询条件有 bug

**建议**: 检查后端 UserViewSet 的认证状态过滤逻辑

---

#### 2. TC-ADMIN-004_非管理员访问
**错误类型**: `AssertionError: 状态码不匹配：期望 403, 实际 200`  
**失败原因**: student 账号访问 admin 接口应该返回 403，但返回了 200

**问题分析**:
- 后端权限验证失效
- student 账号可能被错误授予了 admin 权限
- 或者后端没有正确使用 `IsAdminUser` 权限类

**严重性**: ⚠️ **高** - 涉及系统安全性

---
